# 暗网与 Tor 洋葱路由技术原理

> 一句话定义：暗网是基于覆盖网络构建的匿名通信系统，以 Tor（The Onion Router）为核心协议，通过多层加密和随机中继实现通信双方的匿名性。

## 核心思想

**洋葱路由：** 在不可信的网络基础设施上建立可信的匿名通道。不信任网络中的任何人，信任建立在密码学之上。

基本流程：
1. 客户端选择三个中继节点（入口、中间、出口）
2. 用出口节点的密钥加密→包一层→用中间节点的密钥加密→再包一层→用入口节点的密钥加密
3. 每层像洋葱皮，每个节点只能解密自己那一层
4. **每个节点只知道自己的上家和下家，不知道完整路径**

## 三个概念辨析

| 概念 | 定义 |
|------|------|
| 表层网 | 搜索引擎能索引到的页面 |
| 深网 | 无法被搜索引擎索引的内容（邮箱、企业内部系统） |
| 暗网 | 需要特殊协议才能访问的[[覆盖网络]]（.onion 站点） |

深网 ≠ 暗网。Gmail 收件箱是深网，不是暗网。

## Tor 网络架构

### 节点三种角色

```
客户端 → 入口节点(Guard) → 中间节点(Middle) → 出口节点(Exit) → 目标
  ↑                           ↑                      ↑
 看完整路径                   只知道上下家           知道目标不知源头
```

- **入口节点：** 知道客户端 IP，不知道最终目的地
- **中间节点：** 不知道客户端也不知道目的地
- **出口节点：** 知道目的地和明文内容（如果是 HTTP），不知道客户端

出口节点是最大风险点——如果访问 HTTP 而非 HTTPS，出口节点可看到全部明文。Tor 浏览器默认强制 HTTPS。

### 目录服务器

10 个权威目录服务器维护所有活跃节点列表（共识文档），通过共识机制决定可信节点，客户端从中获取。

## 电路构建与密钥协商

客户端需和三个节点分别做密钥交换，通过层层转发完成：

1. 客户端与入口节点直接做 ntor 密钥交换（基于 Curve25519）→ 生成 K1
2. 客户端通过入口节点转发，与中间节点做密钥交换（内容被 K1 加密）→ 生成 K2
3. 客户端通过两层转发，与出口节点做密钥交换 → 生成 K3

发送数据时，原始数据依次用 K3→K2→K1 加密，每个节点解密一层后转发。

### ntor 协议

- 基于 Curve25519 椭圆曲线 Diffie-Hellman
- 提供[[完美前向保密]]（PFS）：服务器长期密钥泄露也不影响历史会话
- 比 RSA 握手更快，已替代 RSA-1024
- 每个方向各有一个 AES-128-CTR 对称密钥，512 字节 cell 为基本传输单元

## 隐藏服务

暗网网站实现双向匿名，.onion 地址本质是服务端公钥的哈希：

| 版本 | 长度 | 算法 | 状态 |
|------|------|------|------|
| v2 | 16 字符 | SHA-1 + RSA-1024 | 已弃用 |
| v3 | 56 字符 | SHA-3 + ed25519 | 当前 |

六步连接流程：
1. 服务端选择 3 个引入点
2. 将公钥 + 引入点打包成 HS Descriptor 发布到 DHT
3. 客户端查 DHT 获取描述符
4. 客户端选择汇聚点（Rendezvous Point）
5. 通过引入点通知服务端来汇聚点会合
6. 双方在汇聚点建立[[端到端加密]]通信

**最终效果：** 客户端不知道服务器在哪，服务器不知道客户端是谁，汇聚点不知道通信内容，引入点不知道通信在进行。

## 密码学全栈

| 层级 | 算法 | 用途 |
|------|------|------|
| 链路层加密 | AES-128-CTR | 每个中继节点的对称加密 |
| 密钥交换 | ntor（Curve25519 ECDH） | 协商会话密钥 |
| 身份验证 | ed25519 | 节点签名、隐藏服务身份 |
| 端到端加密 | TLS | 客户端与目标服务器 |

体现[[纵深防御]]——任何单一机制被攻破，其他层仍提供保护。

## 流量分析对抗

| 攻击方式 | 对抗手段 |
|---------|---------|
| 时序攻击（关联入口/出口流量） | 固定 512 字节 cell + 随机延迟 |
| 流量指纹（推断访问内容） | obfs4 伪装随机噪声、meek 伪装 CDN 流量 |
| ISP 封锁公开节点 | 桥接节点（未公开 IP + obfs4 混淆） |
| [[深度包检测]]（DPI 识别 Tor 流量） | 可插拔传输（Pluggable Transports） |

**关键限制：** 如果攻击者同时控制入口和出口节点并监控，理论上仍可能去匿名化。

## 另一条路径：[[I2P]]

| 特性 | Tor | I2P |
|------|-----|-----|
| 设计目标 | 匿名访问外网 | 构建内部匿名网络 |
| 路由方式 | 洋葱路由（单向电路） | 大蒜路由（双向隧道） |
| 数据打包 | 按 cell 传输 | 多条消息打包成"大蒜" |
| 出口到明网 | 原生支持 | 不鼓励 |
| 延迟 | 较低 | 较高 |

大蒜路由把多条消息（含回复路径）打包在一个加密"大蒜"中一起发送，增大[[流量分析]]难度，代价是更高延迟。

## 本质总结

暗网没有发明新物理网络。在 TCP/IP 之上构建[[覆盖网络]]——用普通互联网基础设施，改变路由和加密方式。

**核心矛盾：** 如何在不可信的网络中建立可信的匿名通道？
**答案：** 信任建立在密码学之上，而非对网络基础设施的信任。
