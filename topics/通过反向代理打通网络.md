# 通过反向代理打通不通网络的服务

## 问题抽象

有一个服务在域名 A 上，有 `/x` 和 `/y` 两个路径，但 A 的网络不可达。域名 B 网络可达。如何通过 B 访问 A 上的服务？

核心思路：**利用一个可中介的网络节点，做流量搬运**。

## 原理：反向代理做了什么

[[反向代理]]本质上做了三件事：

### 1. 接替握手

客户端的 TCP 连接终止于代理服务器 B，B 与 A 之间**另开一条独立的 TCP 连接**。

```
客户端 TCP ──→ B（反向代理） ──→ 新的 TCP ──→ A
```

关键点：客户端 IP 不会透传到 A（除非代理显式传递 `X-Forwarded-For` 等头），A 只看到 B 的 IP。

### 2. 路径映射

在 [[HTTP]] 层面，反向代理将请求路径从 B 映射到 A 的实际路径：

| 客户端请求 | 代理转发到 |
|-----------|-----------|
| `https://b.com/x` | `https://a.com/x` |
| `https://b.com/x` | `https://a.com/some-other-path/x` |

路径前缀匹配 + 替换是反向代理最基础的功能。

### 3. 协议透传

代理在应用层（HTTP）理解请求，所以可以：
- **改写 Host 头**（B → A），让后端收到正确的 Host
- **添加请求头**（`X-Forwarded-For`, `X-Real-IP` 等）
- **处理 [[WebSocket]] 升级**
- **缓存响应、限制速率**等

## 常见实现方式

### Nginx 反向代理

在 B 的服务器上配置：

```nginx
server {
    listen 443 ssl;
    server_name b.com;

    location /x/ {
        proxy_pass https://a.com/x/;
        proxy_set_header Host a.com;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /y/ {
        proxy_pass https://a.com/y/;
        proxy_set_header Host a.com;
        # ...
    }
}
```

### 前提：B 到 A 的网络必须通畅

单纯在 B 上配反向代理还不够——**B 的服务器必须能访问 A**。这是整个方案的瓶颈。

## 关键瓶颈：B 到 A 的路由不通怎么办？

### 方案 A：B 与 A 在同一内网（最简单）
B 服务器和 A 服务在同一内网/VPC，直接配 `proxy_pass http://内网IP/x/`。最经典的架构。

### 方案 B：隧道穿透（A 主动连出来）
如果 A 没有公网 IP，但能主动出站，用 [[frp]] / ngrok / [[Cloudflare Tunnel]] 这类工具：

```
A ──建立隧道──→ B（公网服务器）
                ↑
    客户端 ──请求 B──┘
```

A 主动发起连接到 B，在 B 上暴露端口。客户端通过 B 访问，流量走隧道回到 A。

### 方案 C：WireGuard VPN
如果 A 和 B 都是服务器，可以建一个 [[WireGuard]] VPN，把 A 和 B 放在同一个虚拟网络里。然后 B 的 `proxy_pass` 直接指向 A 的 VPN 虚拟 IP。

```
A（VPN IP: 10.0.0.2） ←── WireGuard 隧道 ──→ B（VPN IP: 10.0.0.1）
                                               proxy_pass http://10.0.0.2/x/
```

### 方案 D：Cloudflare Tunnel（零信任）
如果 A 能出站但不想暴露任何入站端口，cloudflared 可以让 A 通过 Cloudflare 的边缘网络暴露服务，B 只要域名指向 Cloudflare 即可。

## 一句话总结

[[反向代理]]解决的是**路径映射 + 请求中转**，但能否打通还取决于 **B 到 A 的网络连通性**——如果 B 到 A 也不通，就需要[[隧道穿透]]或 [[VPN]] 把 A 拉入 B 可达的网络空间。
